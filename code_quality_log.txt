--- IMPORTANT: `styler` vs `lintr` ---
This script uses two tools for code quality:
1. `styler`: Reformats your code LAYOUT (spacing, indentation, line breaks around operators).
   It CHANGES your files directly if their layout doesn't match its style guide.
   If `styler` reports 'File unchanged', its layout was already compliant.
2. `lintr`: Checks for a WIDER range of style issues (e.g., variable names, commented code),
   potential bugs, and code complexity. These are often the 'underlines' in your editor.
   `lintr` DOES NOT automatically change your files. This script WILL let you CONFIGURE `lintr`
   to ignore specific checks project-wide (via a .lintr file), which can reduce 'underlines'.
   Otherwise, lintr issues require manual code changes.
